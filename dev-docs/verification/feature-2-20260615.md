## Observations

- The security-critical failure paths (atomic write, old-key preservation on
  `OSStatus` failure) remain unit-only by design: there is no launch arg to arm
  `InMemoryAPIKeyStore.failNextWrite` from a UI test, and injecting Keychain
  faults through the UI would be a weaker signal than the scripted `KeychainOps`
  sequences already covering them.
- All four user-facing criteria now have a pixel-level pass. The only thing
  between feature #2 and VERIFIED is a Release build exercised on a physical
  device — a named, genuine environment gap, not a tooling-laziness deferral.
