# Feature #6 — verification harness (XCUITest target + DebugBridge + UI-test isolation) — verification

The harness is the deliverable. It is verified by exercising every harness
mechanism end-to-end against a running app on the simulator — which is exactly
what the harness exists to enable. No device-only leg: the harness is a Debug
test seam by construction.
