vrecorder-v2 冷启动手册（小白版）
从初始化到让 Claude 自己干活

不需要你懂编程。这份手册用大白话，一步步带你把一个新项目"点火"，最后让 AI 助理在你睡觉时也能自己写代码、自己检查、自己修 bug。每一步都给你照着抄就行的提示词。

👤 这份手册写给谁：完全没有计算机背景、但想用 Claude Code 这类 AI 工具把一个项目"自动跑起来"的人。看到不懂的词别慌，每个词第一次出现都会用生活里的东西打比方。
你会依次做这几件事 👇 0｜先搞懂一件事：你到底在搭什么？ 1｜一次性准备：装好三样工具 2｜地基已经帮你打好了（认识这些文件夹） 3｜让 AI 助理认识这个项目（第一句话） 4｜体验"交作业就自动有人批改"（提交即审计） 5｜给助理一张"任务清单" 6｜按下"无人值守"开关（让它自己连轴转） 7｜第二天早上，你看什么 8｜怎么喊停 & 常见担心

0你到底在搭什么？

一句话：你在雇一个"会自己干活、还会自己检查作业"的 AI 程序员助理。

🧑‍🍳 打个比方：想象你开了一家餐厅，但厨师是个 AI。你不用站在厨房盯着。你只要做三件事：① 把菜谱和账本放在厨房（让它知道做什么、做到哪了）；② 装一个质检员，每道菜出锅前自动尝一口（不合格就退回）；③ 设一个闹钟，定时叫它起来按订单做菜。这三样齐了，你回家睡觉，餐厅自己运转。
📒 账本（记忆）
项目里的文字文件，记着"要做什么、做到哪、为什么这么做"。AI 每次来上班先读它，就知道接着干。
🛂 质检员（审计）
每次 AI 想把新代码"交上去"，自动请另一个 AI独立检查一遍，有严重问题就拦下来。
⏰ 闹钟（定时唤醒）
到点自动把 AI"叫醒"开一局新班，照着账本挑一件活干完，再睡下。
📝 任务清单（台账）
你用大白话写下"我想要这个功能""这里有个毛病"，AI 从清单里挑活做。

好消息：账本、质检员、闹钟的框架，已经在 vrecorder-v2 这个文件夹里帮你搭好了。你主要要做的，是"准备工具 → 让助理认识项目 → 写任务清单 → 按下自动开关"。

1一次性准备：装好三样工具

这一步一辈子只做一次。装完就不用再管。

① Claude Code —— 你的 AI 助理本体

它是一个在"终端"里运行的 AI（终端 = 电脑里那个黑黑的、能打字命令的小窗口，Mac 上叫"终端 Terminal"）。按官网指引装好、用你的 Claude 账号登录即可。

② Codex —— 那位"质检员"

这是另一家公司（OpenAI）的 AI，专门用来独立审查助理写的代码（两个不同的 AI 互相挑错，比一个人自检靠谱）。装好后用订阅账号登录就行，不用填什么密钥/key：

# 在终端里粘贴这两行： npm install -g @openai/codex codex login # 浏览器会弹出来，用你的 ChatGPT 账号登录
✅ 为什么强调"不用 key"：用订阅账号登录，按月付费，便宜很多，也不用保管一串容易泄露的密码。这正是这套项目特意配置好的方式。
③ Git / GitHub 账号 —— 给代码拍快照、存云端

Git = 给你的项目不断拍快照的工具，随时能回到任何一张旧快照。GitHub = 把这些快照存到云端的网站（像代码的网盘 + 相册）。注册个 GitHub 账号，按提示登录即可。

2地基已经帮你打好了

打开 vrecorder-v2 文件夹，你会看到这些。不用改，认识一下就行：

  • AGENTS.md —— 给助理的"员工手册"。AI 一上班先读它，知道这个项目的规矩。
  • docs/ 里的 features.md / bugs.md / tasks.md —— 你的三本台账：想要的新功能、发现的毛病、随手记的杂事。这是你主要要动笔的地方。
  • .claude/ —— 规矩和自动机关都在这（质检规则、定时任务的说明书）。
  • scripts/git-hooks/pre-push —— 那位"质检员"的触发开关。它已经装好：你每次想把代码上传云端时，会自动先审一遍。
  • dev-docs/ —— 项目的长期记忆：计划、决策、验证记录都放这。
⚙️ 只需做一次的小动作：第一次打开项目，在终端里进入这个文件夹，运行一行命令把"质检员开关"接上（如果还没接）：
bash scripts/git-hooks/install.sh
看到 "The Codex audit gate now runs on every 'git push'" 就成功了。

3让 AI 助理认识这个项目

在 vrecorder-v2 文件夹里启动 Claude Code，然后把下面这段话发给它（这就是"提示词"——你对助理说的话）。

提示词 · 第一次见面 你好。请先读这个项目的 AGENTS.md 和 docs/architecture.md，告诉我：① 这是个什么项目？② 你需要遵守哪些规矩（尤其"提交即审计"是怎么回事）？③ 现在项目处于什么阶段、下一步该做什么？用大白话回答，不要用太多术语。
✅ 它读完会用人话给你讲清楚这个项目、规矩和现状。这一步是确认"助理真的看懂了说明书"——以后每开一局新班，它都会自己读，不会失忆。

4体验"交作业就自动有人批改"

这是这个项目最特别的地方，也是你重启它的原因——先亲眼看一次它怎么工作。

让助理做一件很小的事，然后把成果"上传云端"。上传的那一刻，质检员会自动跳出来审查：

提示词 · 先走一遍流程 请做一个最小的改动（比如在 README 里加一句话），然后带我走一遍"提交并上传"的完整流程。当你执行 git push 时，会自动触发 Codex 质检——把质检的结论用大白话讲给我听。如果质检拦住了，告诉我它担心什么、该怎么办。

你会看到：上传前，终端里冒出一行行"正在审查…"，几分钟后给出结论。如果代码有严重问题，上传会被拦下来，并把审查报告存进 .claude/codex-audits/。

🛂 为什么这很重要：传统做法是 AI 写完直接交，错了你事后才发现。这里是"出门前自动安检"——有严重问题根本出不了门。而且从项目第一天就生效，不是以后才补。

5给助理一张"任务清单"

AI 不会凭空知道你想要什么。你得把需求写进台账。用大白话写就行，它会帮你整理成正式条目。

提示词 · 提需求（新功能） 我想加一个功能：「（这里用你自己的话描述，比如：开始翻译时屏幕上方显示一个红色'录音中'的小圆点）」。请把它按规矩整理成 docs/features.md 里的一条，填好"要解决什么问题、范围、要考虑的特殊情况、怎么测、怎么算做完"，并在 GitHub 上建一个对应的条目。
提示词 · 报毛病（bug） 我发现一个问题：「（用你的话描述现象，比如：来电话之后，翻译就不出声了）」。请把它登记到 docs/bugs.md，并建一个对应的 GitHub 条目。先别急着修，只登记。
✅ 台账里有了"待办"的条目，第 6 步的自动模式才有活可干。清单是空的，闹钟叫醒它也只会说"没活干"。

6按下"无人值守"开关

前面都顺了，就可以让它自己连轴转了。两个前提：① 台账里有活（第 5 步）；② 允许它自己干，不用每步都问你同意。

先：先手动陪它完整做完一件事

在放手之前，你先盯着它把第一个功能从头做到尾一次，确认整条流水线（写代码→质检→测试→上传）真的通。这一步是"试运行"。

提示词 · 手动跑通第一件事 请用 /feature-workflow 实现 docs/features.md 里的第 1 个功能。严格按六个步骤来、不要跳步：先写测试、再写代码、然后让 Codex 质检、再验证、最后才上传。每完成一步用一句话告诉我进度。
再：打开自动开关

确认能跑通后，让它进入"无人值守"——像给餐厅装上定时器：

提示词 · 启动无人值守 请帮我启动无人值守工作流：按 .claude/cron-prompts/ 里的说明，定时自动做四件事——验证（verify）、修 bug（bugfix）、做功能（feature）、看门狗（watchdog）。每次自动运行都要在日志里如实记录"开始/结束/结果"，没活干就如实写"没活干"，不要硬编活儿。运行期间常规操作请自动放行、不要每步都停下来等我点同意。
⏰ 关于"闹钟"：定时任务是跟着当前这次运行的，关机或过 7 天会自动停。第二天想继续，重新发一次上面的"启动无人值守"提示词即可。看门狗会帮你续期、清理卡住的任务。

7第二天早上，你看什么

想知道它夜里干了啥，不用懂代码，看三样就够。直接问助理：

提示词 · 早间复盘 用大白话给我一份"昨夜工作汇报"：① 看 .claude/cron-logs/ 的日志，每个定时任务都正常"开始→结束"了吗？做成了什么、有没有报错？② 有没有新上传的成果，每个都通过质检和验证了吗？③ 有没有卡住的任务需要我处理？挑重点讲，别堆术语。
✅ 把这当成每天早上花两分钟听一次"值班汇报"。一切正常就继续；有异常它会指给你看，你再决定。

8怎么喊停 & 常见担心

🛑 想让它停下来？

提示词 · 停请停止所有无人值守的定时任务，并做好交接：告诉我现在到哪一步了、有什么没做完。

😟 常见担心

  • "它会不会乱改、把项目搞坏？" —— 每次上传前有独立质检拦截严重问题；而且每一步都拍了快照（git），随时能退回。
  • "它会不会乱花钱？" —— 质检用的是订阅账号（按月固定费用），不是按次烧钱的密钥。
  • "它会不会自己瞎编需求？" —— 不会。它只从你写进台账的清单里挑活；清单没有的，它不会自作主张做。
  • "我电脑关了它还跑吗？" —— 不跑。定时任务靠你这台电脑上的这次运行；关机就停，开机重新发"启动"提示词即可。
  • "它会不会自己设计界面，结果很丑？" —— 规矩里写死了：界面必须来自你提供的设计稿，它不许自创；没设计稿它会停下来等你给。
🎯 整页总结成一句话：装好工具(第1步)→ 地基已就绪(第2步)→ 让助理读懂项目(第3步)→ 亲眼看一次自动质检(第4步)→ 把需求写成清单(第5步)→ 手动陪跑一遍再按下自动开关(第6步)→ 每天早上听一次汇报(第7步)。顺序别反：先把"做什么"和"怎么把关"准备好，最后才让闹钟开始自动叫醒它。
